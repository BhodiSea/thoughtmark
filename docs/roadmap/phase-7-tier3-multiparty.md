# Phase 7 ‚Äî Tier 3 (TEE/ZKML) + Multi-party (Fabric)

> Optional/deferred computational-provenance layer: prove "model M produced output Y on input X" ‚Äî never faithfulness. **Source:** [architecture.md](../research/architecture.md) ¬ß9 / ¬ß8.4 / ¬ß5.7 / ¬ß16 / ¬ß17, [roadmap.md](../research/roadmap.md) ¬ß2 (Tier 3), ¬ß6 Phase 4, Recommendation 5, Thresholds, Caveats.

## Objective

Attach **computational provenance** to a sealed trail without touching frozen core SemVer or any existing artifact hash. This phase is explicitly **OPTIONAL / DEFERRED** ‚Äî it is roadmap [¬ß6 Phase 4](../research/roadmap.md) and [architecture ¬ß17 Phase 4](../research/architecture.md). It activates two of the seams that were frozen at 1.0 in [phase-3-verify-apifreeze.md](phase-3-verify-apifreeze.md):

- the `Attestor` trait (TEE attestation: NVIDIA NRAS / Phala), populating the predicate's reserved optional `inference_attestation` ([arch ¬ß5.7](../research/architecture.md), [¬ß9](../research/architecture.md)), and
- the deferred `AnchorKind::Fabric` plugin for genuine multi-party-distrust ([arch ¬ß8.4](../research/architecture.md)).

The claim this phase enables is bounded and stated in the bytes: an `AttestationRef` binds "this output came from model M on input X" (computational provenance). It does **NOT** claim chain-of-thought faithfulness or validity-of-record ‚Äî that limit is permanent in the type system (`NotEstablished.faithfulness` / `validity_of_record`, invariant **I7**, [arch ¬ß19](../research/architecture.md)). TEE attestation *moves* the capture-integrity trust boundary; it does not eliminate the oracle problem ([roadmap ¬ß4(a)](../research/roadmap.md), [arch ¬ß19](../research/architecture.md)). ZKML is **reserved** for small, high-stakes classifier/scoring models only and must not block this (or any earlier) phase ([roadmap Recommendation 5](../research/roadmap.md)).

- **architecture ¬ß9 (`thoughtmark-attest-tee`)** ‚Üí the `Attestor` plugin crate; the non-repudiable-vs-attested honesty band surfaced in the UI ([arch ¬ß9.1](../research/architecture.md), [¬ß15](../research/architecture.md)).
- **architecture ¬ß8.4 (Fabric plugin)** ‚Üí `thoughtmark-anchor-fabric` (deferred Phase 4); chaincode keyed by root hash with an N-of-M endorsement policy; orderer-asserted time labeled weaker than OTS/RFC 3161.

These are **already defined in `thoughtmark-core` as part of the Phase 3 1.0 freeze** (the ¬ß14.4 injection-traits-and-proof-types list ships `Attestor` alongside `#[non_exhaustive] AttestationKind` and `#[non_exhaustive] AnchorKind` as empty, populated-later seams ‚Äî see [phase-3-verify-apifreeze.md ‚Üí Deliverables, "Injection traits + proof types"](phase-3-verify-apifreeze.md)). This phase therefore only *activates* pre-existing frozen seams; it defines no new trait or enum in core, which is what keeps every change here additive (MINOR).

## Deliverables

- **`thoughtmark-attest-tee` ‚Äî the TEE `Attestor` plugin crate** (plain `std`, network-bearing; [arch ¬ß3.6 / ¬ß3.2](../research/architecture.md)). Implements the frozen `Attestor` seam `fn kind(&self) -> AttestationKind; fn attest(&self, claim: &[u8]) -> Result<Vec<u8>>` ([arch ¬ß14.4](../research/architecture.md), [¬ß9](../research/architecture.md)). Two backends behind that one seam:
  - **`TeeNras`** ‚Äî NVIDIA Remote Attestation Service (NRAS) JWT path: the GPU CC enclave produces a signed attestation proving which model ran on which input; the plugin collects the NRAS JWT as the **opaque evidence payload**.
  - **`TeePhala`** ‚Äî Phala dstack / Private-ML-SDK path (demonstrated for 70B-class models, e.g. DeepSeek R1 70B in a GPU TEE).
  - The `attest()` output is opaque `Vec<u8>` evidence (the JWT / attestation document); the **core never learns its internal structure** ([arch ¬ß2.2 / ¬ß16](../research/architecture.md)), so NRAS/Phala format evolution stays inside this plugin without touching core SemVer.
- **Populate the predicate's `inference_attestation` ADDITIVELY** (cite [arch ¬ß5.7](../research/architecture.md), [¬ß16](../research/architecture.md)). On an AI/System/Tool turn, the per-turn `RunManifest` (already bound via `run_manifest_ref`, [arch ¬ß5.8](../research/architecture.md)) gains its previously-reserved `inference_attestation: Option<AttestationRef>` where `AttestationRef { tee: String, evidence_cid: String }` ‚Äî `tee` is the kind tag (`"nras" | "tdx" | "sev-snp" | "ezkl"`), `evidence_cid` is the CIDv1 of the off-ledger evidence blob ([arch ¬ß4.6](../research/architecture.md); evidence is content-addressed, never stored on-ledger, **I5**). Because the field is `#[serde(skip_serializing_if = "Option::is_none")]`, a manifest with the attestation **canonicalizes identically to one without it when absent** ‚Äî so adding it later does **not** change any prior `RunManifest` digest, `TurnId`, `trail_root`, Merkle leaf, checkpoint root, or anchor. This is the [¬ß16](../research/architecture.md) "attach without breaking core" rule made concrete.
- **`AttestationKind { TeeNras, TeePhala, ZkmlEzkl }` is `#[non_exhaustive]` ‚Üí MINOR** ([arch ¬ß14.4 / ¬ß16](../research/architecture.md)). Adding these variants to the 1.0-frozen enum is a code-MINOR release of `thoughtmark-core`; downstream verifiers carry a fail-closed default for unknown kinds ([arch ¬ß14.7](../research/architecture.md)). The plugin crate pins a `>=1, <2` core range so a plugin MAJOR never forces a core MAJOR ([arch ¬ß16](../research/architecture.md)).
- **`thoughtmark-anchor-fabric` ‚Äî the deferred Fabric `Anchorer`/`AnchorVerifier` plugin** ([arch ¬ß8.4](../research/architecture.md), deferred from [phase-4-tier2-anchoring.md](phase-4-tier2-anchoring.md)). A Hyperledger Fabric chaincode **keyed by root hash** with an **N-of-M endorsement policy ‚Äî the proof's strength IS the policy**. Verification (the pure injected `AnchorVerifier::verify_anchor`, [arch ¬ß8.3 / ¬ß14.4](../research/architecture.md)) validates the block's **endorsement set against caller-supplied MSP roots** (never fetched by the verifier ‚Äî the verifier-never-fetches rule, [arch ¬ß8.3 / ¬ß19](../research/architecture.md)). Lands as `AnchorKind::Fabric` (already reserved `#[non_exhaustive]`, [arch ¬ß14.4](../research/architecture.md)) ‚Üí MINOR. Anchor + consistency compose exactly as for OTS/RFC 3161 ([arch ¬ß8.5](../research/architecture.md)): one Fabric anchor over a signed checkpoint root amortizes over every entry under it.
- **Fabric time is WEAKER ‚Äî documented and surfaced as such** ([arch ¬ß8.4 / ¬ß19](../research/architecture.md)). Fabric/orderer time is **orderer-asserted**, not an independent timebase like OTS (Bitcoin block header) or RFC 3161 (a TSA). The UI and docs **MUST NOT present Fabric time as equivalent to OTS/RFC 3161**; `verify()` reports the Fabric anchor's `existed_at_or_before` with its weaker provenance distinguished in `VerificationResult.checks` ([arch ¬ß11.3](../research/architecture.md)).
- **Optional ZKML ‚Äî RESERVED, not built into the headline path** ([roadmap ¬ß2 Tier 3, Recommendation 5](../research/roadmap.md)). `ZkmlEzkl` (EZKL Halo2/ONNX) ‚Äî or Lagrange DeepProve ‚Äî is reserved for **small, high-stakes classifier/scoring models only**; frontier-scale **full** inference proofs remain infeasible (overhead, the ~0.5‚Äì2% quantization accuracy penalty from IEEE-754‚Üífinite-field conversion, practical model-size ceilings ‚Äî [roadmap ¬ß2 hard limits](../research/roadmap.md)). The default Tier-3 mechanism is **TEE attestation**; do **not** block on ZK. The `ZkmlEzkl` variant exists in the frozen enum so the seam is ready, but no shipping ZKML backend is required to exit this phase.

## Quality gates (must be green)

This phase ACTIVATES (it does not re-stand-up the whole net from [phase-0-foundations.md](phase-0-foundations.md)):

- **`cargo-semver-checks` + `cargo-public-api` confirm the additions are MINOR, not MAJOR** (qf Domain 2, API-stability row). Adding `AttestationKind::{TeeNras, TeePhala, ZkmlEzkl}` and exercising `AnchorKind::Fabric` against the 1.0 baseline must report **only additive (MINOR) changes** to `thoughtmark-core`; any MAJOR-classified diff fails CI. This is the mechanical proof that the [Phase 3 API freeze](phase-3-verify-apifreeze.md) survives Tier 3.
- **`byte-parity` / additive-attach regression** (qf Domain 4, conformance-corpus rows): a vector pair proving a `RunManifest` (and its containing `Turn`/`Trail`) produces **byte-identical canonical output and an identical digest** whether `inference_attestation` is absent or `None` ‚Äî the concrete check that [¬ß16](../research/architecture.md) additivity holds (existing artifact hashes unchanged). Run through both Rust and WASM/TS engines.
- **Attestation-evidence verification tests** (qf Domain 6, authoritative-vectors row): NRAS-JWT and Phala-attestation parse/verify fixtures; Fabric endorsement-set validation against supplied MSP roots, including a negative vector where the endorsement set fails the N-of-M policy ‚Üí fail-closed.
- **A fuzz target per new parser** (qf Domain 2, fuzzing row; activation rule "a fuzz target per new parser"): a fuzz target over the Fabric endorsement-proof/block parser and over the attestation-evidence decode path, in the out-of-workspace `fuzz/` ([ADR-0006](../research/architecture.md)). These join the cid/jcs (P1) and merkle/dsse/did_key/anchor-proof (P2/P4) targets.
- **Honesty checks ‚Äî docs/UI never present computational provenance as faithfulness** (invariant **I7**, [arch ¬ß19](../research/architecture.md); qf honesty frame). A machine-checkable doc/string gate that (a) Tier-3 attestation is described as "model M produced output Y on input X," never as reasoning faithfulness or validity; (b) the verifier UI keeps the **non-repudiable vs attested** band separation ([arch ¬ß9.1 / ¬ß15](../research/architecture.md)) ‚Äî "signature valid (cryptographic)" vs "claims attested by issuer/TEE Z"; (c) Fabric time is labeled weaker than OTS/RFC 3161 and never rendered equivalent ([arch ¬ß8.4 / ¬ß19](../research/architecture.md)).

## Exit / freeze gate

- **TEE attestation reference works end-to-end:** a captured AI turn gets a real NRAS-or-Phala attestation, its evidence is content-addressed (`evidence_cid`), the `RunManifest`'s `inference_attestation` is populated additively, and the trail still verifies with **identical hashes** to the un-attested run.
- **Core stays 1.x** ([arch ¬ß17 Phase 4](../research/architecture.md)) ‚Äî every change here is a code-MINOR (`AttestationKind` / `AnchorKind` variants) and a plugin release; no wire-format or canon change, so no new `canon_version`. The [wire-format freeze (end of Phase 2)](phase-2-log-signing.md) and the [API/1.0 freeze (end of Phase 3)](phase-3-verify-apifreeze.md) both remain intact and unviolated.
- **ZKML stays optional.** Nothing in this phase's exit depends on a shipping ZKML backend.
- **THRESHOLD that flips the plan ([roadmap Thresholds](../research/roadmap.md)):** if DeepProve/EZKL reach **Llama-scale full-inference proofs at <1000√ó overhead**, **PROMOTE ZKML from optional to a headline Tier-3 feature** (DeepProve already lists Llama-class models in active development ‚Äî watch this closely). Until then, ZKML remains the reserved seam and TEE is the default.

## Risks & open implementation-gaps

- **Vendor ZKML benchmarks need independent verification** ([roadmap Caveats](../research/roadmap.md)). DeepProve / zkPyTorch figures come substantially from vendor sources (Lagrange, Polyhedra) and must be independently benchmarked before being relied on. DeepProve's "first full LLM" claim is **GPT-2-scale (124M)**; **Llama-class proving is described as in development, not shipping**. zkPyTorch's published Llama-3 8B figure (~150s/token, 99.32% cosine similarity) is a per-token research result, not a turnkey product. Do not over-promise in docs or UI.
- **TEE moves the trust boundary but does not eliminate the oracle problem** ([roadmap ¬ß4(a)](../research/roadmap.md), [arch ¬ß19](../research/architecture.md)). A genuine TEE attestation binds "model M ran on input X," but garbage-in is still immutably preserved and the schema must keep recording fields honestly as `model_self_reported_*` ([arch ¬ß5.7](../research/architecture.md)). Never let a green TEE check be read as a claim about content truth or faithfulness (**I7**).
- **Fabric time is orderer-asserted** ([arch ¬ß8.4 / ¬ß19](../research/architecture.md)) ‚Äî strictly weaker than OTS (Bitcoin) / RFC 3161 (TSA). The ever-present implementation gap is UI/doc presentation: it must surface the weaker provenance, never equivalence.
- **No frozen NRAS/Phala/EZKL evidence wire format in the ecosystem** ‚Äî vendor attestation formats evolve. The architecture's mitigation is structural: the `Attestor` payload is **opaque `Vec<u8>`** and the verifier consumes evidence via injected seams, so format churn lands in the plugin without a core MAJOR ([arch ¬ß2.2 / ¬ß16](../research/architecture.md)). Pin plugin dependencies low and add real-service conformance fixtures as the SDKs stabilize.
- **ZKML quantization penalty** (~0.5‚Äì2% accuracy from IEEE-754‚Üífinite-field, [roadmap ¬ß2 hard limits](../research/roadmap.md)) plus float-on-the-hashed-path being banned (**I4**) means any ZKML output bound on-ledger must commit only integer/fixed-point or content-addressed artifacts ‚Äî the reserved `ZkmlEzkl` seam must respect the no-float rule like every other hashed path.
