# Phase 1 — Tier 0 (Canonicalization, Hashing & Content Addressing) + determinism runtime

> The deterministic byte foundation every higher tier hashes, Merkle-izes, signs, and anchors — bytes in, bytes/`Result` out, no clock, no RNG, no network. **Source:** [architecture.md §4 (4.1–4.9)](../research/architecture.md), [§10.1–§10.2](../research/architecture.md), [§3.5–§3.6](../research/architecture.md), [§13.2](../research/architecture.md), [§2 invariants](../research/architecture.md); [quality-foundations.md Domain 2 + Domain 4](../research/quality-foundations.md).

This phase is **architecture [§17](../research/architecture.md) "Phase 0 — Tier 0 core"**: it lands [§4](../research/architecture.md) (`canon`/`hash`/`cid`, `tm-jcs-1`) and the determinism runtime [§10](../research/architecture.md), backs `spec/vectors` v0.x with **real** Tier-0 output (not stubs), and turns the cross-language `byte-parity` gate from "green against stubs" into "green against the actual canonicalizer". **Nothing is frozen** — the workspace stays `0.x` (the wire-format freeze is [phase-2-log-signing.md](phase-2-log-signing.md), the API/1.0 freeze is [phase-3-verify-apifreeze.md](phase-3-verify-apifreeze.md)).

## Objective

Implement the single Tier-0 contract — **identical logical input MUST yield byte-identical canonical output in Rust core, WASM, and TS** ([I1](../research/architecture.md)) — and the injected-determinism runtime that makes higher tiers reproducible. Concretely: stand up the two audited crates (`thoughtmark-core`, `thoughtmark-schema`) with their preamble and no-panic lint wall, build the `canon/` module set (JCS choke point, no-float walker, `Digest`/`HashAlg`, domain separation + `CANON_VERSION`, CIDv1, salted commitments, Tier-0 error model), the `determinism/` traits (`Clock`/`Rng`/`Csprng` + `UnixMillis`), the crate-wide flat `Error`/`ErrorCode`, and the first real `spec/vectors/` corpus (canon/hash/cid + negative). All of this is pure ([P2](../research/architecture.md)): no `SystemTime::now`, no `thread_rng`, no socket, no float on the hashed path.

## Deliverables

- **`thoughtmark-core` + `thoughtmark-schema` crate skeletons** with the [§3.6](../research/architecture.md) `lib.rs` preamble verbatim: `#![no_std]` + `extern crate alloc` (and `#[cfg(feature = "std")] extern crate std`), `#![forbid(unsafe_code)]` (inherited from `[workspace.lints.rust]`), `#![deny(missing_docs)]`, and the no-panic clippy wall `#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic, clippy::indexing_slicing, clippy::arithmetic_side_effects, clippy::unreachable, clippy::todo, clippy::float_arithmetic, clippy::string_slice)]`. One core + one schema crate, not a `-canon`/`-crypto`/`-merkle` split ([ADR-0002](../research/architecture.md); [§3.7](../research/architecture.md)). Workspace deps pinned to the [§3.6](../research/architecture.md) budget (`serde_json` with `arbitrary_precision`, `serde_json_canonicalizer = "0.3"`, `blake3`/`sha2`/`cid`/`multihash`/`subtle`/`thiserror`, all `default-features = false`). (arch [§2.3](../research/architecture.md), [§3.6](../research/architecture.md))
- **`canon/mod.rs`** — Tier-0 public surface (re-exports of the items below). (arch [§4.1](../research/architecture.md))
- **`canon/jcs.rs`** — THE choke point over `serde_json_canonicalizer`; nothing else in the workspace calls that crate directly. Exposes `canonicalize<T: Serialize>(&T) -> Result<Vec<u8>, CanonError>`, `canonicalize_value(&serde_json::Value)`, and `canonicalize_str(&str)` (the WASM/TS-boundary entry; `arbitrary_precision` ON so large ints survive parse, then `validate_no_float` runs). Pin the four RFC 8785 behaviors in vectors — most critically the **UTF-16 code-unit key sort** (`u16`-compared, NOT Rust `char`/UTF-8 byte order, NOT code-point order); we never sort keys ourselves. (arch [§4.2](../research/architecture.md))
- **`canon/nofloat.rs`** — `validate_no_float(&serde_json::Value) -> Result<(), CanonError>` rejecting any `f64` Number or any integer outside the I-JSON safe range `[-(2^53-1), 2^53-1]`. **Run BEFORE `canonicalize`** on all hashed data. (arch [§4.3](../research/architecture.md))
- **`canon/digest.rs`** — `#[non_exhaustive] HashAlg { Blake3, Sha256 }` with `multihash_code()` (blake3 = `0x1e`, sha2-256 = `0x12`), `digest_len()` = 32, and `as_str()` wire tokens `"blake3" | "sha256"` (NOT `"sha2-256"`). `Digest { alg, bytes: [u8; 32] }` (`Copy`, no alloc) with `to_hex()` + `multihash_bytes()`. **Hand-written `Serialize`/`Deserialize`** (NOT derive) emitting exactly `{"alg":"blake3","bytes_hex":"<64 lowercase hex>"}`; fail-closed deserialize (unknown `alg` → `UnknownHashAlg`; `bytes_hex` length ≠ 64 → `CanonInvalidJson`). `hash_with(alg, &[u8]) -> Digest` and `hash(&[u8]) = hash_with(Blake3, ·)`. (arch [§4.4](../research/architecture.md))
- **`canon/domain.rs`** — `pub const CANON_VERSION: &str = "tm-jcs-1"` (the format identifier, bound INTO every preimage; one value everywhere — [ADR-0001](../research/architecture.md)); the **three** content-hash domains `OBJECT` = `"thoughtmark.object"`, `TURN` = `"thoughtmark.turn"`, `MANIFEST` = `"thoughtmark.manifest"`; `prefix(alg, domain) -> Vec<u8>` building `CANON_VERSION ":" alg.as_str() ":" domain ":"` (e.g. `b"tm-jcs-1:blake3:thoughtmark.turn:"`); and the core primitive `hash_domain(alg, domain, canonical_json) -> Digest` = `H(prefix || canonical_json)`. These are the **only three** structured-JSON objects that get a self-identifying id — everything else is a salted content digest of raw bytes ([§4.7](../research/architecture.md)), and the log leaf/node hashes are RFC 6962 ([§6](../research/architecture.md)), deliberately distinct. (arch [§4.5](../research/architecture.md))
- **`canon/cid.rs`** — CIDv1 for opaque binary blobs (never inlined into canonical JSON). `pub const RAW_CODEC: u64 = 0x55`; `cid_blob(alg, blob) -> Result<Cid, CanonError>` (multihash over the **raw** digest bytes, NOT JCS/domain-prefixed); `cid_to_string(&Cid) -> String` pinning base32-lower (multibase `b`) — never `Cid::to_string()`'s default. Build via `Multihash::wrap(code, digest_bytes)` with hard-coded codec constants; **pin BLAKE3 multihash length to 32 and reject any parsed CID whose BLAKE3 multihash length ≠ 32** (0x1e is variable-length). Shares the length-32 encoding with `digest.rs::multihash_bytes()` (a cross-module invariant covered by a CID vector). (arch [§4.6](../research/architecture.md))
- **`canon/salt.rs`** — `Salt(pub [u8; 32])` (supplied by the caller's INJECTED RNG; core never calls `rand`, [I3](../research/architecture.md)) and `salted_content_digest(alg, &Salt, content) -> Digest` = `hash_with(alg, salt.0 || content)` over **raw** bytes (NOT JCS, NOT domain-prefixed). The salt is **off-ledger by construction** — it is NOT part of the digest's serialization and NEVER enters a canonical `Turn`/signed `Statement`; this is the redaction/crypto-shredding foundation ([§9](../research/architecture.md)) and the reason `ContentDigest::Hashed` carries `digest_hex` only, never `salt_hex` ([ADR-0012](../research/architecture.md), [I5](../research/architecture.md)). (arch [§4.7](../research/architecture.md))
- **`canon/error.rs`** — `CanonError` (`thiserror`-derived, `no_std` via `core::fmt`, **content-free** messages): `InvalidJson`, `FloatNotAllowed`, `IntegerOutOfRange`, `DuplicateKey`, `Multihash`, `Cid`, `UnknownHashAlg`, `UnknownCanonVersion`. Maps into the crate-wide `Error`/`ErrorCode` via the [§10.2](../research/architecture.md) table. (arch [§4.9](../research/architecture.md))
- **`determinism/` module** — the injection traits `Clock { fn now(&self) -> UnixMillis }`, `Rng { fn fill_bytes(&mut self, dst: &mut [u8]) }`, and the marker `Csprng: Rng` (crypto-grade only). `UnixMillis(i64)` with a **custom `Serialize`/`Deserialize` emitting/accepting a decimal STRING** (`"1750000000000"`) — never a JSON number — so values beyond 2⁵³ never violate [I4](../research/architecture.md)/I-JSON and round-trip through JS `BigInt`; internally an `i64` for integer comparisons. The deterministic, `vectors`-gated impls: `FixedClock(pub UnixMillis)`, `SeededRng` (ChaCha20, `impl Rng` only — deliberately NOT `Csprng`), and `#[cfg(feature = "vectors")] VectorCsprng(SeededRng)` (`impl Csprng`, gated so production can never mint a real salt/key from a test seed). `SystemClock`/`OsCsprng` live ONLY outside the audited core. (arch [§10.1](../research/architecture.md); feature gates [§3.5](../research/architecture.md))
- **Feature gates** wired per [§3.5](../research/architecture.md): core default features = `["std"]` only (wasm path builds `--no-default-features --features alloc`); `keygen` (off by default) will later gate the one RNG-injecting signer helper; `vectors` (off by default) gates `FixedClock`/`SeededRng`/`VectorCsprng`; `json_schema` on `thoughtmark-schema` (off by default; only `-schemagen` enables it) gates the `schemars` derives so production/wasm never carry `schemars`. CI runs `cargo hack --feature-powerset` to prove every combination builds. (arch [§3.5](../research/architecture.md))
- **`error.rs` — the crate-wide flat error model** ([§10.2](../research/architecture.md)): one `#[non_exhaustive]` `thiserror` `Error` whose `Display` strings are **constant and content-free** (crypto failures collapsed into one `Signature` Display, distinguishable only by code — no oracle), plus `Error::Internal(&'static str)` for every "impossible" case (a static site tag, never runtime/secret data — because a Rust panic crossing WASM becomes an uncatchable `RuntimeError`). The single `#[non_exhaustive] #[serde(rename_all = "SCREAMING_SNAKE_CASE")] ErrorCode` set (append-only, never renumbered: the Tier-0/runtime-relevant codes `CanonInvalidJson`, `CanonNonDeterministicFloat`, `CanonIntegerOutOfRange`, `UnknownCanonVersion`, `UnknownHashAlg`, `DigestMismatch`, `CidMalformed`, … `Internal`), `Error::code()`/`static_message()`, and `pub type Result<T>`. Implement the **per-tier `CanonError` → `ErrorCode` → TS string → vector token** mapping table (`FloatNotAllowed → CanonNonDeterministicFloat → "CANON_NON_DETERMINISTIC_FLOAT"`, `IntegerOutOfRange → CanonIntegerOutOfRange`, `UnknownCanonVersion → UnknownCanonVersion`, …). (arch [§10.2](../research/architecture.md), [§2.3](../research/architecture.md))
- **`spec/vectors/` v0.x — real Tier-0 corpus** generated once from native Rust (`--bless`) then frozen as raw byte files ([§13.1](../research/architecture.md)): `canon/NNNN/` (`input.json` → `expected.bin`), `hash/NNNN/` (`input.json` → `blake3.hex` + `sha256.hex`), `cid/NNNN/` (`input.bin` → `expected.txt`), and `negative/NNNN/` (`input.json` → `expected_code.txt`) covering at minimum **NaN, ±Inf, duplicate keys, integer > 2⁵³, and an astral-plane (U+10000+) key**. `manifest.json` declares each case's `op`/`canon_version = "tm-jcs-1"`/`alg`/`env`/expected file(s) and the toolchain pin (`rustc 1.96.0`, `wasm_bindgen 0.2.125`, `wasm_opt: disabled`); negative vectors assert **both engines return the same `ErrorCode`** (fail-closed). The `conformance` job now runs against REAL Tier-0 output. (arch [§13.2](../research/architecture.md); qf Domain 4 / bootstrap row 10)
- **`thoughtmark-testkit` scaffold** (dev-only, `std`, not published): loads `spec/vectors` and drives core; shared by the native `cargo nextest run -p thoughtmark-core --test conformance` oracle and (later) the TS executors. (arch [§13.3](../research/architecture.md); directory tree [§3.2](../research/architecture.md))
- **`thoughtmark-schemagen` scaffold** (build-only, `std`, not published): generates `schemars` JSON Schema + TS types from `thoughtmark-schema` behind the `json_schema` feature; stubbed here (schema types fill in at [phase-2-log-signing.md](phase-2-log-signing.md)), present so the crate graph and feature gating are exercised from day one. (arch [§3.2](../research/architecture.md), [§3.5](../research/architecture.md))

## Quality gates (must be green)

This phase **activates** (turns from green-against-stubs to green-against-real-code) the following — it does not re-stand-up the whole Phase 0 net:

- **`byte-parity` (cross-language differential), now over real Tier-0** — native Rust oracle vs WASM-under-Node, WASM-under-Chromium/Firefox/WebKit, **and** the independent pure-TS oracle (`cyberphone/canonicalize` + `@noble/hashes` + `multiformats`), each reproducing every canon/hash/cid/negative vector byte-for-byte. The pure-TS oracle exists specifically to guard the **UTF-16 JCS key sort** ([§4.2](../research/architecture.md)) — the exact divergence that killed `serde_jcs`. (qf Domain 4 / bootstrap row 10; arch [§13.1](../research/architecture.md), [§13.3](../research/architecture.md))
- **`proptest`** — encode the Tier-0 invariants as properties: `canonicalize` is **idempotent**, `hash(canonicalize(x))` is **stable**, and `validate_no_float` **rejects every negative-vector class** (NaN/±Inf/>2⁵³); run in both engines. (qf Domain 2 / bootstrap row 8; arch [§4.2–§4.3](../research/architecture.md))
- **Fuzz targets `jcs` + `cid`** — the first two per-parser `cargo-fuzz` targets (the activation rule: a fuzz target per new parser as it appears; `merkle`/`dsse`/`did_key` follow at [phase-2-log-signing.md](phase-2-log-signing.md), anchor proofs at [phase-4-tier2-anchoring.md](phase-4-tier2-anchoring.md)). Run in the nightly `fuzz/` stage (outside the workspace — [ADR-0006](../research/architecture.md)), native and WASM-under-`wasmtime`, asserting equality. (qf Domain 2 / bootstrap row 9; arch [§13.4](../research/architecture.md), [§3.1](../research/architecture.md))
- **`clippy::float_arithmetic` + `disallowed-methods`/`disallowed-types`** — the no-float wall and the nondeterminism bans (`SystemTime::now`, `Instant::now`, `thread_rng`, `rand::random`, `serde_json::to_vec`/`to_string` on hashed data; `SystemTime`/`f32`/`f64` types scoped to `canon`/`hash`/`cid`/`merkle`) now bite against real code in `canon/`. (qf Domain 2 / bootstrap rows 5, 6, 11; arch [§2.1 I2/I3/I4](../research/architecture.md))
- **The `cargo-tree` CI assertion** — `cargo build -p thoughtmark-core --target wasm32-unknown-unknown --no-default-features --features alloc` must prove core's closure contains **no `getrandom`, no `wasm-bindgen`, no `tokio`, no `reqwest`** (and the `cargo metadata` check that no networking/plugin crate appears among core's dependents). This is the structural guard against the single most insidious failure mode. (arch [§2.2 P1](../research/architecture.md), [§2.3](../research/architecture.md))

**ADRs that bite here:** [ADR-0001](../research/architecture.md) (JCS crate = `serde_json_canonicalizer`; `serde_jcs` banned — record at [phase-0-foundations.md](phase-0-foundations.md), enforced here by the choke point + `disallowed-methods`), [ADR-0002](../research/architecture.md) (one core + separate schema), [ADR-0012](../research/architecture.md) (salt external to the hashed leaf).

**Tier-0 vectors green** across all four executors (native Rust, WASM/Node, WASM/3 browsers, pure-TS oracle), with every negative vector returning the matching `ErrorCode` fail-closed, the `cargo-tree` no-leak assertion passing, and `jcs`/`cid` fuzz targets running clean. **NOTHING is frozen — the workspace is `0.x`** ([§17](../research/architecture.md): "Nothing frozen (0.x)"). The wire-format freeze candidate is [phase-2-log-signing.md](phase-2-log-signing.md); the API/1.0 freeze is [phase-3-verify-apifreeze.md](phase-3-verify-apifreeze.md). `CANON_VERSION` is already `"tm-jcs-1"` and bound into preimages, so artifacts produced now stay verifiable under [§16](../research/architecture.md)'s evolution rule (add `canon_v2`, never mutate `canon_v1`; unknown version → `UnknownCanonVersion`, fail closed).

## Risks & open implementation-gaps

- **Feature-unification leaking `getrandom`/`js` (or `ed25519-dalek/rand_core`) into the audited core** — explicitly "**the single most insidious failure mode**" ([§2.2 P1](../research/architecture.md)): it silently re-introduces host nondeterminism and breaks [I1](../research/architecture.md)/[I3](../research/architecture.md). Mitigation: the wasm-target `cargo-tree` assertion + `cargo metadata` dependents check + `cargo hack --feature-powerset`; `getrandom` is confined to `thoughtmark-wasm` only ([§3.6](../research/architecture.md)).
- **`serde_json` coercing big integers to `f64` before `validate_no_float` can reject them** — mitigated by enabling `serde_json`'s **`arbitrary_precision`** so a large integer from stored JSON survives the parse and reaches the no-float walker, which then rejects it as `IntegerOutOfRange`/`CanonIntegerOutOfRange` ([§4.3](../research/architecture.md), [§3.6](../research/architecture.md)).
- **UTF-16 JCS key-sort divergence** — astral-plane keys sort differently under UTF-16 code units vs code-point/UTF-8 order; this is the exact class that killed `serde_jcs`. We never sort keys ourselves, pin an astral-plane vector, and guard with the independent pure-TS oracle ([§4.2](../research/architecture.md), [§13.3](../research/architecture.md)).
- **Variable-length BLAKE3 multihash (0x1e)** — two CIDs for the same content could differ if length is not pinned; mitigated by hard-pinning length 32 and rejecting any parsed CID with BLAKE3 length ≠ 32, with the shared length-32 encoding covered by a cross-module CID vector ([§4.6](../research/architecture.md)).
- **`Digest` derived-serde trap** — a derived serde would emit `bytes` as a 32-number JSON array (reopening the no-array-of-ints concern) and not match every example/vector; the hand-written impl emitting `{"alg","bytes_hex"}` is load-bearing and easy to regress ([§4.4](../research/architecture.md)).
- **Panic-across-WASM** — any `unwrap`/`expect`/`panic`/indexing/arithmetic-overflow in core becomes an uncatchable `RuntimeError` at the boundary; the no-panic clippy wall plus `Error::Internal(&'static str)` for "impossible" cases is the structural defense, but it must be held in every new function ([§2.3](../research/architecture.md), [§10.2](../research/architecture.md)).
- **`spec/vectors/` is the oracle, blessed once from native Rust** — any later expected-byte change is by definition a breaking corpus release ([§16](../research/architecture.md)/[P4](../research/architecture.md)); blessing must be deliberate, and pure-TS-oracle drift is investigated, never auto-blessed ([§13.1](../research/architecture.md), [§13.3](../research/architecture.md)).
