# Phase 0 — Foundations & Quality Spine

> Stand up the *entire* gate net against stub crates so the first implementation PR already runs inside the wall. **Source:** [quality-foundations.md](../research/quality-foundations.md) Domains 1–8, Part 2 config appendix, the 4-stage bootstrap order, the 27-row `[MUST]` checklist; [architecture.md](../research/architecture.md) §2 (invariants I1–I8 as constraints), §3 (workspace, crate graph, directory tree, §3.6 manifests, §3.7 ADR seeds), §18 (day-one ADRs).

## Objective

Build the *pre-implementation quality contract*: everything that must exist **before the first line of implementation code**, so the first impl PR cannot merge unless it satisfies the deterministic, machine-checkable gates. The defining constraint ([qf §0](../research/quality-foundations.md)) is that ~100% of this codebase is authored by Claude Code with no human reviewing every line, so quality rests on gates the agent cannot rationalize past, "temporarily" disable, or silently weaken. The governing principle is **hooks are advisory, CI is authoritative — every hook/local check is mirrored by a required CI check** ([qf §0](../research/quality-foundations.md)); a control present in only one layer is not-yet-real. This phase produces no product behavior: it produces the net. Exit is achieved against **stub crates** (functions that return "not implemented"), and the single most load-bearing control — the cross-language conformance gate (Domain 4 / row 10) — must already be running, even if it only asserts the stubs agree on "not implemented."

The honesty frame is inherited verbatim and is load-bearing on the structure, not just the prose: the system proves **integrity-of-record** (existed at T, in lineage L, unaltered since capture), never **validity-of-record** or **faithfulness** (I7, [arch §2.1](../research/architecture.md)). Phase 0 records this as a first-class artifact ([`docs/threat-model.md`](../research/quality-foundations.md), row 22) so green CI can never be misread as a claim about the content being notarized.

- **SHA-pinned, hardened Actions** (row 15; [qf Domain 5](../research/quality-foundations.md)) — every third-party `uses:` pinned to a full commit SHA (not a tag), verified by **`zizmor`** (security static-analysis), **`pinact`** (pin enforcement), and **`actionlint`** (syntax + shellcheck); StepSecurity **`harden-runner`** for egress auditing and least-privilege `permissions:` blocks. CI shape in [qf Part 2 ci.yml](../research/quality-foundations.md).
- **Multi-OS / multi-Rust matrix incl. `wasm32-unknown-unknown`** (rows 4, 14; [qf Domain 5](../research/quality-foundations.md)) — `stable` + MSRV `1.96.0` (+ nightly allowed-to-fail) via `dtolnay/rust-toolchain` and `Swatinem/rust-cache`, including a `cargo build -p thoughtmark-core --target wasm32-unknown-unknown --no-default-features --features alloc` job that asserts `getrandom` and `wasm-bindgen` are **absent** from `cargo tree` (the feature-unification leak in [arch P1/§2.2](../research/architecture.md) is the most insidious failure mode).
- **Required-checks gate** (row 17; [qf Domain 5](../research/quality-foundations.md)) — the wall: `cargo fmt --check`, `clippy -D warnings`, `cargo nextest run`, the **conformance/determinism job**, `cargo deny check`, `cargo audit`, `tsc --noEmit`, `biome check`, `wasm-bindgen-test`, `publint`/`attw`. Merging is impossible until green.
- **Repository rulesets** (row 16; [qf Domain 5](../research/quality-foundations.md)) — not legacy branch protection: require the named status checks, **signed commits**, linear history, PR review, and protect **release tags** (immutable). Immutable signed history is part of the provenance trust model the project itself sells.
- **The dependency-direction CI assertion** ([arch §3.3](../research/architecture.md), I8) — a script over `cargo metadata` that fails the build if (a) `thoughtmark-core`'s reverse-deps include any `*-anchor-*`/`*-identity*`/`*-attest*`/`*-c2pa`/`*-log`/`*-redaction`/`wasm` crate, or (b) core's dependency closure includes any networking crate, `getrandom`, `wasm-bindgen`, `tokio`, or `reqwest`. Wired now against stubs (**ADR-0004** keeps the graph legible for this check).
- **Audited crypto crates pinned + key-hygiene discipline as design rules** (rows 19, 20; [qf Domain 6](../research/quality-foundations.md), I6) — pin `ed25519-dalek` 2.x (**`verify_strict` always** — the only sanctioned verification path; a clippy lint bans bare `verify`), `sha2`, `blake3` in `[workspace.dependencies]` (all `default-features = false`; **no `rand_core` in core**, [arch §3.6](../research/architecture.md)); encode `subtle` (constant-time eq/select), `zeroize` (wipe key material on `Drop`), `secrecy` (`Secret<T>`) as design rules from commit zero. - **Reproducible builds** (row 23; [qf Domain 7](../research/quality-foundations.md), [arch §3.6](../research/architecture.md)/§12.5) — commit `Cargo.lock` and build `--locked --frozen`; set `SOURCE_DATE_EPOCH`; `[profile.release] trim-paths = "all"` (RFC 3127, stable since 1.81 — **never** hand-roll `--remap-path-prefix`); **pin the `wasm-opt`/binaryen version** (its output is not byte-stable across versions). ## Quality gates (must be green)

Phase 0 is unique: **it stands up the ENTIRE gate net at once, against stubs** ([qf bootstrap order](../research/quality-foundations.md), the quality-gate activation rule). Later phases *activate* gates as the code they target first appears; here every gate must exist and run green against stub crates. The gates newly standing up in this phase:

- **The conformance/determinism job** (row 10, ⭐; [qf Domain 4](../research/quality-foundations.md)) — present and running from the first PR, asserting Rust⟷WASM/TS byte-equality (against stubs: both sides agree on "not implemented").
- **Rust gate** (rows 4, 5, 6, 8, 11; [qf Domain 2](../research/quality-foundations.md)) — `cargo fmt --check`, `clippy --all-targets --all-features -- -D warnings` (the no-panic wall + nondeterminism + no-float bans), `cargo nextest run --locked`, on the multi-OS/multi-Rust + `wasm32-unknown-unknown` matrix.
- **Supply-chain gate** (rows 7, 21; [qf Domain 6](../research/quality-foundations.md)) — `cargo deny check`, `cargo audit`, `gitleaks`, the dependency-direction `cargo metadata` assertion ([arch §3.3](../research/architecture.md)).
- **Actions-hardening gate** (row 15; [qf Domain 5](../research/quality-foundations.md)) — `zizmor` + `pinact` + `actionlint`, `harden-runner` egress audit.
- **TS/WASM gate** (rows 12, 13, 14; [qf Domain 3](../research/quality-foundations.md)) — `tsc --noEmit`, `biome check`, `wasm-bindgen-test` (Node + headless browser), `publint`/`attw`, `knip`.
- **Docs/licensing gate** (rows 24, 26; [qf Domain 8](../research/quality-foundations.md)) — `reuse lint`, the SPEC req-ID ↔ vector traceability check.
- **Local mirror** (row 17; [qf Domain 5](../research/quality-foundations.md)) — `lefthook.yml` pre-commit/pre-push run the identical fmt/clippy/typos/biome gates, and `just ci` runs the exact CI graph locally. Every gate above exists in both a local/agent layer (hook) **and** an authoritative CI layer.

Gates *wired but not yet authoritative* (they have no target code yet, so they activate in later phases per the activation rule): Wycheproof/RFC-6962 vectors + Miri activate at [Phase 2](phase-2-log-signing.md); a fuzz target per new parser activates as that parser lands (`jcs`, `cid` at [Phase 1](phase-1-tier0-core.md); `merkle`, `dsse`, `did_key` at [Phase 2](phase-2-log-signing.md); anchor proofs at [Phase 4](phase-4-tier2-anchoring.md)); `cargo-mutants`/`insta` activate as logic lands; `cargo-semver-checks` + release provenance become authoritative at the [Phase 3](phase-3-verify-apifreeze.md) API/1.0 freeze.

## Exit / freeze gate

Phase 0 has **no wire-format or API freeze** (those land at the end of [Phase 2](phase-2-log-signing.md) and [Phase 3](phase-3-verify-apifreeze.md) respectively). Its exit criteria:

- **`just ci` is green against the stub crates** — the whole CI graph passes with crates that return "not implemented."
- **Every one of the 27 `[MUST]` bootstrap rows is satisfied**, each pointing at the file(s) that satisfy it ([qf bootstrap checklist](../research/quality-foundations.md), rows 1–27).
- **The repository ruleset is enforced** (required checks, signed commits, linear history, tag protection — row 16).
- **The conformance job is present and running** (row 10), even if it only checks that the stubs agree on "not implemented" — it must exist *before* any implementation PR.
- **Harness lock-down is in place** (row 2) — the agent cannot edit `.claude/**`, `.github/**`, `deny.toml`, or `rust-toolchain.toml`, and `managed-settings.json` requires a human for guardrail edits.
- **Day-one ADRs recorded** — **ADR-0001** (JCS crate `serde_json_canonicalizer`; `serde_jcs` BANNED), **ADR-0002** (one `thoughtmark-core` + separate `thoughtmark-schema`), **ADR-0003** (`just` + pnpm, no Turborepo in v1), **ADR-0004** (no umbrella meta-crate; explicit per-plugin deps), **ADR-0006** (`fuzz/` outside the workspace) — per [arch §18](../research/architecture.md).

The exit handoff to [Phase 1](phase-1-tier0-core.md): the first Tier-0 implementation PR (canon/hash/CID) lands *inside* this net, with the conformance corpus ready to receive its first real expected-byte vectors and the `jcs`/`cid` fuzz targets ready to activate.

## Depends on

None. Phase 0 is the root of the canonical phase index (deps: none). It is the prerequisite for every later phase, beginning with [Phase 1 — Tier 0 — Canon/Hash/CID + determinism](phase-1-tier0-core.md). See the [Roadmap Overview](00-overview.md).

## Risks & open implementation-gaps

- **The agent could weaken its own guardrails.** The primary author is an AI; if it can edit the rules that constrain it, the rules are theatre ([qf Domain 1, harness lock-down](../research/quality-foundations.md)). **Mitigation:** the lock-down `deny` rules + org-level `managed-settings.json` (`allowManagedPermissionRulesOnly` / `strictPluginOnlyCustomization`), and — decisively — **CI is authoritative**: hooks are advisory and can be bypassed, but the required-checks ruleset (row 16) is the wall the agent cannot route around.
- **The conformance gate + lock-down MUST exist before any impl PR.** If implementation begins before row 10 (the cross-language byte-identity gate) and row 2 (lock-down) are live, the most load-bearing control and the guardrails are retrofitted, not enforced — exactly the failure mode this phase exists to prevent ([qf §0](../research/quality-foundations.md), closing note).
- **Feature-unification leakage into the `no_std` core.** A stray `getrandom/js` or `ed25519-dalek/rand_core` enabled by a downstream crate would silently re-introduce host nondeterminism into the audited core ([arch P1/§2.2](../research/architecture.md)). **Mitigation:** the `wasm32-unknown-unknown` build job + `cargo tree` assertion + the dependency-direction `cargo metadata` check, all wired now against stubs; `cargo hack --feature-powerset` proves all combinations build.
- **`sandbox.*` / `managed-settings` / Claude Code config schema may have drifted.** [qf](../research/quality-foundations.md) flags these as "confirm against current docs" (assembled June 2026). **Mitigation:** re-verify the exact key names at setup; treat permissions + sandbox as defense-in-depth, not either/or.
- **`wasm-opt` output is not byte-stable across versions.** Reproducible-build claims for the `.wasm` artifact depend on pinning binaryen ([qf Domain 7](../research/quality-foundations.md), [arch §12.5](../research/architecture.md)). **Mitigation:** pin (or disable) `wasm-opt` and assert the committed `.wasm` BLAKE3 in the rebuild-and-diff job; full turnkey reproducibility is still maturing upstream (`rust-lang/rust#129080`) — verify determinism on Linux CI where the gaps are smallest.
- **Stubs can pass gates vacuously.** A stub that returns `Err(NotImplemented)` satisfies the net without exercising real logic; `cargo-mutants` and most vectors only become meaningful as logic lands (the activation rule). This is acceptable and intended for Phase 0 — the gates' *existence and wiring* is the deliverable; their *teeth* arrive with the code they target in [Phase 1](phase-1-tier0-core.md)+.
- **Cross-tool memory portability.** `CLAUDE.md` is the source of truth; if `AGENTS.md` portability is wanted, make it a symlink/thin pointer rather than a fork ([qf Domain 1](../research/quality-foundations.md)) to avoid two diverging codebooks.
