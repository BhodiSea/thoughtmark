# Phase 2 — Tier 1 Log + Signing + Schema + WASM + full Conformance

> Stand up the whole append-only-log-and-signing wire format and prove it byte-identical across four executors — **the wire-format freeze candidate**. **Source:** [architecture.md](../research/architecture.md) §5 (schema), §6 (Merkle log), §7 (DSSE/in-toto/keys/bundle), §12 (WASM boundary), §13 (conformance harness), §17 (phase map, this is its "Phase 1"); [quality-foundations.md](../research/quality-foundations.md) Domain 4 (cross-language harness), Domain 6 (crypto controls).

## Objective

Turn the deterministic byte foundation from [Phase 1](phase-1-tier0-core.md) into a complete, signed, logged, self-verifying wire format and lock it down with the full 4-executor byte-parity harness. By the end of this phase the `Provenance/v1` schema, the RFC 6962/9162 Merkle log, the DSSE/in-toto signing envelope, the `ThoughtmarkBundle`, and the byte-identical WASM/TS facade all exist and every authoritative vector is green across native Rust, WASM-under-Node, WASM-under-three-browsers, and an independent pure-TS oracle. This is the [architecture.md](../research/architecture.md) §17 **Phase 1** row: its freeze gate is *"Freeze candidate: the wire format."* Nothing in [`verify()`](phase-3-verify-apifreeze.md) or Tier 2 anchoring is built here; this phase makes the bytes that those phases will freeze and verify.

The honesty frame (I7) is load-bearing on every type added here: the schema carries `attested_at`/`attributed_to`/`model_self_reported_version` and an explicit `ApprovalScope` *inside the hashed bytes* ([architecture.md](../research/architecture.md) §5.4), and the signing layer proves only non-repudiation-of-signer, never validity or faithfulness ([architecture.md](../research/architecture.md) §19).

### Merkle log math — `thoughtmark-core::merkle` + `core::checkpoint`

- **`rfc6962.rs`** — `TreeHash([u8;32])` newtype (**always SHA-256**, carries no alg tag, distinct from the Tier-0 content `Digest` — ADR-0013); `LEAF_PREFIX = 0x00` / `NODE_PREFIX = 0x01`; `hash_leaf` (`SHA-256(0x00 || leaf)`), `hash_children` (`SHA-256(0x01 || l || r)`), `empty_root` (`SHA-256("")`). The 0x00/0x01 prefixes are the *only* tree domain separation — never the §4.5 ASCII-string scheme ([architecture.md](../research/architecture.md) §6.1, ADR-0013).
- **`tree.rs`** — `merkle_tree_hash(&[TreeHash])` with the **strict-largest-power-of-two-less-than-n** split (`k = 1 << (BITS-1 - (n-1).leading_zeros())` for n>1; a naive `n/2` is wrong and passes only on powers of two); `TreeState { size, root }` ([architecture.md](../research/architecture.md) §6.2).
- **Iterative verifiers** — `verify_inclusion` (RFC 9162 §2.1.3.1) and `verify_consistency` (RFC 9162 §2.1.4.2, dual-recompute of both old and new roots) — **iterative, constant stack** (so recursion never blows the WASM stack), explicit path-length checks (reject both too-long and too-short proofs — closes the proof-padding forgery vector), `subtle::ConstantTimeEq` for hash equality, take `&TreeHash` not `&[u8]` (so "forgot to domain-hash" is a type error), never panic, never allocate unboundedly ([architecture.md](../research/architecture.md) §6.2).
- **Proof construction** — `InclusionProof`/`ConsistencyProof` types; `NodeId { level, index }` + `MerkleReader` trait (injected node source); `inclusion_proof`, `consistency_proof`, and recompute-only `append` (no persistence; SHA-256 fixed, **no `HashAlg` param**) ([architecture.md](../research/architecture.md) §6.3).
- **`tiles.rs`** — C2SP `tlog-tiles` parser (height-8 / 256-hash / 8192-byte full tiles; big-endian `uint16` length-prefixed entry bundles; the `x`-prefixed 3-digit-group index encoding `1234067 → x001/x234/067`) + a `HashReader` so a thin verifier fetches only O(log n) tiles and verifies **offline** ([architecture.md](../research/architecture.md) §6.5).
- **`core::checkpoint`** — C2SP signed-note Signed-Tree-Head: `Checkpoint { origin, size, root, extensions }`, `checkpoint_body` (deterministic body bytes), `sign_checkpoint`, `verify_checkpoint`. Two exactness traps the vectors pin: the signature line prefix is **em-dash + space** (`— `, U+2014 0x20), not a hyphen; and `verify_checkpoint` must assert **≥1 signature line actually matched** (the note spec ignores unknown sigs, so a "verified" checkpoint could otherwise carry zero valid ones). Key-hash = `SHA-256(keyname || 0x0A || 0x01 || pubkey32)[..4]` ([architecture.md](../research/architecture.md) §6.4).

### Signing & bundle — `core::dsse`, `core::sign`, `core::did_key`, `core::anchor` (seam), `core::bundle`

- **`core::dsse`** — `pae(payload_type, body)` building `"DSSEv1" SP LEN(type) SP type SP LEN(body) SP body` with `LEN` via **`itoa`** (allocation-free, locale-free, no leading zeros — NOT `format!`); `body` = raw JCS-canonical Statement bytes (NOT base64); `DSSE_PAYLOAD_TYPE = "application/vnd.in-toto+json"`. `DsseEnvelope { payload, payload_type, signatures }` / `EnvSig { keyid, sig }`. Encoding rule: **emit STANDARD padded base64 on write**, but **accept both standard and url-safe on read** (the signature is over PAE bytes, independent of the base64 choice) ([architecture.md](../research/architecture.md) §7.1, §7.2).
- **`core::sign`** — `Signer` trait (the seam log/anchor/checkpoint code use so key material never enters those layers); `TmSigner` holding a 32-byte seed in **`secrecy` 0.10 `SecretBox`**, zeroized on drop, never `Debug`/`Display`/`Serialize`/`Clone`; `verify`/`verify_envelope` using **`ed25519-dalek` `verify_strict` ALWAYS** (a clippy `disallowed-methods` lint bans bare `verify` — rejects small-order `A`/`R`, enforces canonical `S`, kills Ed25519 malleability, I6, ADR-0007 one sig/turn); `subtle::ConstantTimeEq` for secret-adjacent comparison, `Zeroizing` intermediates. **`keygen` feature gates the only RNG** (`generate_signer<R: CryptoRng>`) so the audited verify path never links a CSPRNG ([architecture.md](../research/architecture.md) §7.3, §7.4, ADR-0007).
- **`core::did_key`** — the pure ~60-LOC offline `did:key` decoder: strip `did:key:`, require multibase `z`, base58btc-decode, assert the Ed25519 multicodec varint `0xed 0x01`, take exactly 32 bytes, validate on-curve via `VerifyingKey::from_bytes`; malformed/short/off-curve **fails closed** (never a silently-unusable key). On the byte-identity-critical path, vendored into core and covered by oracle vectors (ADR-0010); `did:web`/VC resolution stays out until [Phase 6](phase-6-identity-redaction.md) ([architecture.md](../research/architecture.md) §7.5, §9.1, ADR-0010).
- **`core::anchor` — bare seam types + trait only (no parsers, no backends)** — the bundle's `anchors: Vec<AnchorReceipt>` field already depends on `AnchorReceipt` *here*, so the seam types it names are defined in this phase (core defines only types/trait — no DER/CMS/OTS/Bitcoin-header parsing logic, no deps beyond `serde`; ADR-0008): `AnchorReceipt { schema: "thoughtmark.anchor.receipt/v1", kind: AnchorKind, status: AnchorStatus, root: TreeHash, checkpoint_ref: CheckpointRef, proof: Vec<u8> }` (`#[non_exhaustive]`; `proof` is **OPAQUE** to core, the structured fields are advisory caches re-derived at verify time, never trusted); `AnchorKind` `#[non_exhaustive]` `OpenTimestamps/Rfc3161/Fabric` (adding a variant is a MINOR); `AnchorStatus` `Pending { calendar_uris: Vec<String> }`/`Anchored { time_lower_bound: Option<UnixMillis> }`; `AnchorVerdict` `Valid { time_lower_bound, source: AnchorKind }`/`Pending`/`Invalid(ErrorCode)`; `CheckpointRef` (`origin` + `tree_size` + signer keyid — re-binds a `root` to its log); and the `AnchorVerifier` trait (`fn verify_anchor(&self, checkpoint_bytes, receipt, params) -> AnchorVerdict`) that [`verify()`](phase-3-verify-apifreeze.md) later injects as `&dyn AnchorVerifier`. Only the type/trait *seam* lands here; the async `AnchorBackend`, the **pure** `AnchorVerifier` impl (the parsers), and the concrete `thoughtmark-anchor{,-ots,-rfc3161,-fabric}` backends are [Phase 4](phase-4-tier2-anchoring.md) ([architecture.md](../research/architecture.md) §8.2, §8.3, §7.6; ADR-0008).
- **`core::bundle`** — `ThoughtmarkBundle` self-verifying offline container: `media_type = "application/vnd.thoughtmark.bundle.v1+json"`, `bundle_version: u16` (schema gate → `BundleVersionUnsupported`), `canon_version` (must equal the predicate's → `UnknownCanonVersion`), `envelope` (the DSSE-wrapped Statement — the `ContributionLedger` lives **INSIDE** the signed `predicate`/`Trail`, never a top-level bundle field), `verification_material` (keys/stapled DID docs to resolve each keyid OFFLINE), `inclusion`, `checkpoint` bytes, optional `consistency`, and `anchors: Vec<AnchorReceipt>` (the `core::anchor` seam type defined above; empty until [Phase 4](phase-4-tier2-anchoring.md) populates it). This is the **one** bundle definition; [`verify()`](phase-3-verify-apifreeze.md) consumes it ([architecture.md](../research/architecture.md) §7.6, §11).

### Log shell — `thoughtmark-log` (std; never imported by core)

No Trillian/Tessera for a single tenant; the DDL itself lands with the reference app in [Phase 5](phase-5-reference-app.md) ([architecture.md](../research/architecture.md) §6.5).
- **`TileStorage`** — serves the exact C2SP `tlog-tiles` layout from object storage for the public-log export ([architecture.md](../research/architecture.md) §6.5).
- **Witnesses & monitors are DEFERRED to [Phase 4](phase-4-tier2-anchoring.md)** (split-view defense, `thoughtmark-monitor`); only the storage seam + drivers land here ([architecture.md](../research/architecture.md) §6.6).

### WASM & TS facade — `thoughtmark-wasm` + `packages/core`

- **`thoughtmark-wasm`** (`cdylib`+`rlib`) — the **ONLY** crate that overrides `forbid(unsafe_code)` and the **ONLY** `getrandom/js` quarantine; `#[wasm_bindgen]` thin adapters + the single `glue.rs` `to_js(Error) -> JsError` shim encoding canonical `{code,message}` JSON. The boundary is a **bytes-in/bytes-out airlock**: only `Uint8Array`/`string`/`bigint` cross the line — never a structured JS object, never an `f64` on a byte-significant path. JS `now` (f64) is narrowed to `i64 UnixMillis` with NaN/Inf/fractional rejection at the boundary; `u64`→`bigint` (defeats the 2^53 truncation hazard). The `rlib` lets the §13.4 native differential tests link it ([architecture.md](../research/architecture.md) §12.1–§12.3, §12.6, §10.3).
- **`packages/core` facade** — a single `--target web` artifact (no `--target nodejs`; one byte-audited binary), memoized async `init()`, runtime-specific `node.ts`/`browser.ts` loaders; the facade **never** `JSON.stringify`/`Number.toString`/sorts keys (ESLint-banned in the hashing modules); `package.json` validated by `publint` + `attw` ([architecture.md](../research/architecture.md) §12.2, §12.4).
- **Reproducible `.wasm`** — pin `rust-toolchain.toml` (1.96.0, `wasm32-unknown-unknown`, **not** WASI); **`wasm-bindgen` CLI version MUST equal the crate version exactly** (`=0.2.125`, matching `manifest.json`); **disable `wasm-opt` in v1** (`wasm-opt = false` — its output is not byte-stable across binaryen versions); `SOURCE_DATE_EPOCH` + pinned container + deterministic strip; minimize wasm features (portable BLAKE3, SIMD/reference-types/externref off); CI asserts the committed `.wasm` **BLAKE3** equals an expected hash ([architecture.md](../research/architecture.md) §12.5; [quality-foundations.md](../research/quality-foundations.md) Domain 7, row #23).

### Full conformance harness — `spec/vectors/` + 4 executors ([quality-foundations.md](../research/quality-foundations.md) Domain 4)

## Exit / freeze gate

**WIRE-FORMAT FREEZE CANDIDATE.** The exit condition is the [architecture.md](../research/architecture.md) §17 "Phase 1" gate: the wire format is a freeze candidate. Concretely, all of the following format-identifier *values* and shapes are stable, and every vector is green across **all four executors**:

- schema `predicateType = "https://thoughtmark.dev/Provenance/v1"`;
- `canon_version = "tm-jcs-1"` (bound inside every preimage; unknown fails closed);
- DSSE `payloadType = "application/vnd.in-toto+json"`;
- bundle `media_type = "application/vnd.thoughtmark.bundle.v1+json"` (+ `bundle_version` schema gate).

This is the first of the three freeze gates ([architecture.md](../research/architecture.md) §16): from here, changing any hashed byte requires a new format-identifier *value* + a MAJOR corpus release (add `canon_v2`, never mutate `canon_v1`). It precedes — and is a precondition for — the API/1.0 freeze in [Phase 3](phase-3-verify-apifreeze.md). The schema, log math, signing envelope, and bundle defined here become immutable except by the §16 evolution rule.

## Risks & open implementation-gaps

- **`wasm-opt` output is not byte-stable across binaryen versions.** The conformance gate hashes the committed `.wasm`, so any optimizer drift would break the BLAKE3 assertion. v1 mitigation: **disable `wasm-opt`** (`wasm-opt = false`) and ship the deterministic `wasm-bindgen` output (core is small/pure); re-enable later only via a vendored, checksum-pinned binaryen with a frozen pass list ([architecture.md](../research/architecture.md) §12.5; [quality-foundations.md](../research/quality-foundations.md) Domain 7).
- **Recursion can blow the WASM stack on large trees.** Use the RFC 9162 **iterative** `verify_inclusion`/`verify_consistency` (constant stack) rather than the recursive RFC 6962 forms; pin the choice with large-tree vectors ([architecture.md](../research/architecture.md) §6.2).
- **Ed25519 malleability if bare `verify` is used.** Curve25519's cofactor 8 admits small-order/torsion components and non-canonical `S`, yielding two valid signatures per `(key, message)` — which corrupts Merkle-leaf stability and consistency proofs. Mitigation: **`verify_strict` ALWAYS**, lint-enforced, boundary pinned by Wycheproof + ed25519-speccheck (I6, ADR-0007) ([architecture.md](../research/architecture.md) §7.3).
- **The strict-largest-power-of-two split is a classic Merkle bug.** A naive `n/2` split passes only on powers of two; pin non-power-of-two `merkle_tree_hash` vectors ([architecture.md](../research/architecture.md) §6.2).
- **Signed-note exactness traps.** The signature-line prefix is em-dash + space (U+2014 0x20), not a hyphen; `verify_checkpoint` must require ≥1 *matched* signature (the spec mandates ignoring unknown sigs). Both are pinned by `checkpoint/` vectors ([architecture.md](../research/architecture.md) §6.4).
- **PostgresStorage gap-free sequencing is a hard requirement.** A gap or duplicate index silently corrupts every later root; the advisory-lock sequencer + an optimistic `size` guard must be exercised under concurrent-append tests (full DDL exercised in [Phase 5](phase-5-reference-app.md)) ([architecture.md](../research/architecture.md) §6.5, §15.2).
- **Two-build divergence.** Building a separate `--target nodejs` artifact would risk two non-identical binaries; build **one** `--target web` artifact and select the loader in JS ([architecture.md](../research/architecture.md) §12.2).
- **Shared-compiler blind spot.** Rust and WASM share a compiler, so the independent **pure-TS oracle** (executor D) is what actually guards the UTF-16 JCS sort and a shared-codegen bug; treat its drift as a finding to investigate, never to auto-bless ([architecture.md](../research/architecture.md) §13.3, §4.2).
