# Phase 3 — Verify Pipeline + API Freeze → 1.0

> The offline `verify()` orchestrator lands, the public surface is frozen, and the 1.0 trio ships together with verifiable provenance. **Source:** [architecture.md §11](../research/architecture.md) (verify pipeline), [§14](../research/architecture.md) (frozen API surface), [§16](../research/architecture.md) (versioning trains), [§17](../research/architecture.md) (this is architecture §17 "Phase 2 — API FREEZE → 1.0"); [quality-foundations.md Domain 5](../research/quality-foundations.md) (CI/CD & release automation) and [Domain 7](../research/quality-foundations.md) (release provenance & supply chain).

## Objective

Compose every Tier-0/Tier-1 primitive into the single offline `verify()` orchestrator (a pure function over the bytes stapled in a `ThoughtmarkBundle`, no socket opened), encode the integrity-of-record honesty frame **permanently in the type system** as `Established` / `NotEstablished`, and then **freeze the entire public API surface** — the 10 verbs, their types, the extension-seam traits, and the four format-identifier *values* — and publish the **1.0 trio** (`thoughtmark-core` + `@thoughtmark/core` + `thoughtmark-vectors`/`@thoughtmark/vectors`) together with OIDC trusted publishing, `--provenance`, and SLSA attestations. After this phase the surface is immutable except by the [§16](../research/architecture.md) versioning rules. This is architecture [§17](../research/architecture.md) Phase 2.

## Deliverables

- **`core::verify` orchestrator** ([arch §11.2](../research/architecture.md)) — `pub fn verify(bundle: &ThoughtmarkBundle, policy: &Policy, clock: &dyn Clock, anchors: &dyn AnchorVerifier) -> VerificationResult` in the `verify/` module of `thoughtmark-core` (see crate map [arch §3](../research/architecture.md), `src/.../verify/`). Reads the clock **exactly once** (`now = clock.now()`) at entry; runs each check **independently** (one failure never masks another) in the fixed, byte-stable `CheckKind` order: `BundleSchema → CanonVersion → DsseSignature → StatementBinding → MerkleInclusion → Checkpoint → Consistency (skipped if no proof) → AnchorReceipt (skipped if !require_anchor) → ContributionLineage`. Each check pushes a `CheckOutcome`; `total` is the AND of all REQUIRED checks (`Skipped` is neutral).
- **`Policy` input** ([arch §11.1](../research/architecture.md)) — `#[serde(deny_unknown_fields)]` struct carrying the caller's assertions: `expected_subject_digest`, `trusted_keys`, `log_origin`, `trusted_log_keys`, `required_witnesses: u8` (k-of-n cosignatures), `require_anchor: bool`, `max_clock_skew_ms: i64`, `required_actions: Option<Vec<Action>>`, `accepted_canon_versions: Vec<CanonVersion>`.
- **Per-check semantics** ([arch §11.2](../research/architecture.md)) — **DsseSignature**: rebuild PAE, `verify_strict` each `EnvSig` against `trusted_keys` (resolved via stapled `verification_material`), decode payload to `Statement`/`Trail`. **StatementBinding**: `subject.digest == trail_root` (and `expected_subject_digest` if supplied); `subject.name` carries the `tree_size` matching `inclusion.tree_size`. **MerkleInclusion/Checkpoint**: `verify_inclusion(inclusion, checkpoint.root)`; `verify_checkpoint` signed by `trusted_log_keys` with ≥ `required_witnesses` cosignatures. **Consistency**: `verify_consistency` when a proof is present. **AnchorReceipt**: `anchors.verify_anchor(checkpoint, receipt, params)` per receipt with `time_lower_bound ≤ now + max_clock_skew_ms`. **This check is wired but inert at 1.0:** the orchestrator calls the injected `&dyn AnchorVerifier` exactly as documented, but **core ships no `AnchorVerifier` impl** — the DER/CMS/OTS/Bitcoin-header parsers live in `thoughtmark-anchor`, which is delivered in [Phase 4](phase-4-tier2-anchoring.md) (ADR-0008, [arch §8.3](../research/architecture.md)). Until a real impl is injected, `require_anchor` runs against a no-op/none verifier, so **Phase 3 alone cannot establish `existed_at_or_before`** (the tightest UPPER time bound is the min over *passing* anchors, of which there are none without a Phase-4 verifier); `Established.existed_at_or_before` stays `None` and the honesty frame's `time_upper_bound_only` note holds. The seam, the `CheckKind::AnchorReceipt` slot, and the `Established.existed_at_or_before` field are all frozen here; the *capability* to populate that bound arrives with the Phase 4 `AnchorVerifier`.
- **`VerificationResult` / `CheckOutcome` / `Established` / `NotEstablished`** ([arch §11.3](../research/architecture.md)) — the result types, all `#[serde(deny_unknown_fields)]`. `VerificationResult { schema: "thoughtmark.verification_result/v1", verified_at: UnixMillis (the injected now), total: bool, checks: Vec<CheckOutcome> (fixed order), established, not_established }`. `Established { existed_at_or_before (tightest UPPER bound = min over passing anchors), unaltered_since_capture (= DsseSignature && StatementBinding && MerkleInclusion && Checkpoint), lineage, bound_subject_digest, signed_by (signer keyids/DIDs), log_origin }`. **`NotEstablished` is CONSTANT in v1** — the integrity-of-record honesty frame (I7) baked into the type system, always present with `&'static str` fields `validity_of_record`, `faithfulness`, `authorship_truth`, `completeness`, `time_upper_bound_only`. **`verify()` returns a VALUE, never `Err`, for a well-formed run** ([arch §10.3 / §11.3](../research/architecture.md)): a tamper is a *successful run with `total = false`*, so the full proven/not-proven report survives to the UI; `Err`/`JsError` is reserved for malformed input (bad JSON, out-of-range clock, unsupported bundle version).
- **ContributionLineage DAG validation** ([arch §11.2](../research/architecture.md)) — the final check walks the `Trail`'s `parents` DAG and asserts: **no cycles**, **no dangling parent**, **every `supersedes` target exists**, `attested_at` **non-decreasing** along the chain, the `RunManifest` digest **matches** `run_manifest_ref`, and `policy.required_actions` are **present** (e.g. an AI `create` + a human `approve`). This is the lineage validation the schema's `LEDGER_BROKEN_LINK` / `LEDGER_NON_MONOTONIC_TIME` codes back; populates `Established.lineage` (a `Vec<LineageStep>`) only when it passes.
- **The FROZEN [§14](../research/architecture.md) public API surface** — frozen as *the verbs + their types + the extension-seam traits + the format-identifier values*:
  - **10 verbs** ([arch §14.1](../research/architecture.md)): `canonicalize`, `hash` (Tier 0, core/pure); `append` / `inclusion_proof` / `verify_inclusion`, `consistency_proof` / `verify_consistency` (Tier 1, core/pure); `sign` / `verify_envelope` (core/pure over injected `Signer`); `verify_anchor` (in `thoughtmark-anchor`, pure, injected into `verify`) and `anchor` (submit — **async/shell**, **not a core fn**, ADR-0008); `verify` ([§11](../research/architecture.md), core/pure → `VerificationResult`); `redact` (`thoughtmark-redaction`, pure, proof-bearing digests unchanged); `export` (core/pure, lossy view, **never re-hashed**).
  - **4 format-identifier VALUES** ([arch §14.2](../research/architecture.md), frozen as values not crate versions): `canon_version = "tm-jcs-1"` (bound INTO the hashed preimage; fail-closed on unknown), `predicateType = "https://thoughtmark.dev/Provenance/v1"`, DSSE `payloadType = "application/vnd.in-toto+json"`, bundle `media_type = "application/vnd.thoughtmark.bundle.v1+json"` (+ `bundle_version: u16` schema gate).
  - **Core types** ([arch §14.3](../research/architecture.md)): `CanonVersion` (`TmJcs1`, fail-closed `parse`), `HashAlg` (`Blake3`/`Sha256`), `Digest` (custom serde `{"alg","bytes_hex"}`), `TreeHash([u8;32])`, `UnixMillis(i64)` (decimal-string serde), the single `#[non_exhaustive]` `Error`/`ErrorCode` set, plus the §5 schema types reused without a second definition (`Trail`, `Turn`, `ContributionLedger`, `Action`, `ApprovalScope`, `Participant`, `RunManifest`, …).
  - **Injection traits + proof types** ([arch §14.4](../research/architecture.md)): `Clock`, `Rng`/`Csprng`, `Signer`, `MerkleReader`, `AnchorVerifier`, `IdentityResolver`, `Attestor`; `TreeState`, `InclusionProof`, `ConsistencyProof`, `#[non_exhaustive] AnchorKind` / `AnchorReceipt`, `#[non_exhaustive] AttestationKind`. All time/RNG injected; `#![forbid(unsafe_code)]`, `#![deny(missing_docs)]`.
    - **These seams must be *defined in core at/before this phase* — freezing presupposes their existence** ([arch §14.4](../research/architecture.md), [§16](../research/architecture.md): the `Attestor`/`Anchorer`/`AnchorVerifier`/`Signer`/`MerkleReader`/`IdentityResolver` seams are *"frozen as `#[non_exhaustive]` traits at 1.0"*). The 1.0 freeze cannot freeze types that do not yet exist, so the type *shapes* — `AnchorVerifier`, `AnchorKind`, `AnchorReceipt`, `Attestor`, `AttestationKind` (and `AttestationRef`) — are **declared in `thoughtmark-core` here**, even where their behaviour is supplied later. The later phases do **not** introduce these types: [Phase 4](phase-4-tier2-anchoring.md) only ships an `AnchorVerifier` *impl* (in `thoughtmark-anchor`) and exercises the `AnchorKind` slots, and [Phase 7](phase-7-tier3-multiparty.md) only *populates* the predicate's `inference_attestation` and adds `AttestationKind` *variants* against the already-frozen enum — both purely additive (MINOR) precisely because the frozen 1.0 surface already contains the empty seams.
    - **The empty `Attestor` seam ships now** ([arch §14.4](../research/architecture.md)): the trait `Attestor { fn kind(&self) -> AttestationKind; fn attest(&self, claim: &[u8]) -> Result<Vec<u8>>; }`, the `#[non_exhaustive] enum AttestationKind { TeeNras, TeePhala, ZkmlEzkl }`, and the reserved predicate type `AttestationRef { tee: String, evidence_cid: String }` ([arch §5.7](../research/architecture.md)) are all **defined in core at the 1.0 freeze** — *empty until [Phase 7](phase-7-tier3-multiparty.md)*, which is the only phase that wires an `Attestor` impl (`thoughtmark-attest-tee`) and populates `inference_attestation: Option<AttestationRef>`. No phase ≤ 3 other than this one creates them, so the surface would otherwise freeze a hole.
  - **TS mirror `@thoughtmark/core`** ([arch §14.6](../research/architecture.md)): one-to-one with the Rust verbs; `u64`→`bigint` everywhere (never `number`); digests/proofs are `Uint8Array`; version/alg enums are string-literal unions; the **`ErrorCode` union is EXACTLY EQUAL to the Rust set**. Two documented boundary mappings: `verify` takes `nowUnixMs: bigint` + an optional `anchorVerifier` instead of trait objects, and `export_` is the reserved-word rename (still never re-hashed).
- **Versioning trains wired into CI** ([arch §16](../research/architecture.md)):
  - **Code SemVer** — `cargo-semver-checks` + a `cargo-public-api` snapshot gate `thoughtmark-core`; `api-extractor` report + `changesets` gate `@thoughtmark/core`; `release-plz` integrates `cargo-semver-checks` to block accidental breaks ([qf Domain 5](../research/quality-foundations.md)).
  - **Corpus SemVer** — `spec/vectors/` → `thoughtmark-vectors` / `@thoughtmark/vectors`; **any expected-byte change = MAJOR**; the corpus is a **`dev-dependency`** so a MAJOR forces an explicit reviewed bump in both engines; `byte-parity` CI enforces it.
  - **Format identifiers** — the `release-train-consistency` CI check **refuses any PR that mutates an expected byte string without introducing a new format-identifier value** + a MAJOR corpus release.
  - **Forward-verifiability** — `verify()` dispatches on the **embedded** `canon_version` to a frozen `canon_vN` module (never deleted; deprecate-and-warn at most); an **unknown version FAILS CLOSED** (`UNKNOWN_CANON_VERSION`, a permanent negative vector, since a best-effort recompute under guessed rules could forge a PASS). Add `canon_v2`, never mutate `canon_v1`.
- **Publish the 1.0 trio TOGETHER** ([arch §16/§17](../research/architecture.md); [qf Domain 7](../research/quality-foundations.md)) — `thoughtmark-core 1.0.0` + `@thoughtmark/core 1.0.0` + `thoughtmark-vectors` / `@thoughtmark/vectors 1.0.0` released as one event; **pin the toolchain in `manifest.json`** (asserted-equal to CI by `byte-parity`). Release via **crates.io Trusted Publishing (OIDC)** and **npm Trusted Publishing (OIDC) + `npm publish --provenance`** — **no long-lived `NPM_TOKEN`/PAT** — with **GitHub Artifact Attestations** (SLSA build provenance via Sigstore), **SBOMs** (`cargo-cyclonedx` + `@cyclonedx/cyclonedx-npm`) attached, and a **consumer-side verify** step (`npm audit signatures`, `cosign verify`) to dogfood the mission.

## Quality gates (must be green)

This phase **activates** (per the activation rule — only the new gates, not the whole net):

- **`cargo-semver-checks` + `cargo-public-api` snapshot (Rust) and `api-extractor` report + `changesets` (TS)** gate **every PR** — accidental SemVer breaks now fail CI ([qf Domain 5 release automation row](../research/quality-foundations.md); [arch §16](../research/architecture.md)).
- **`verify/*` conformance vectors** added to `spec/vectors/` with **byte-parity** asserted native vs WASM — the JCS-canonical bytes' hash of the `VerificationResult` is pinned ([arch §11.3 / §13](../research/architecture.md)).
- **`cargo-mutants` on the verify logic** — the orchestrator and each check must be mutation-covered so a flipped comparison or skipped check is caught ([qf Domain 4 mutation-testing, activated here as the logic lands](../research/quality-foundations.md)).
- **`release-train-consistency` check** — refuses a byte mutation without a new format-identifier value + MAJOR corpus release ([arch §13.4 / §16](../research/quality-foundations.md), [qf Domain 5](../research/quality-foundations.md)).
- **Release provenance & signing** — GitHub Artifact Attestations (SLSA), `npm publish --provenance`, OIDC trusted publishing (crates.io + npm, no long-lived tokens), **OpenSSF Scorecard** action + Best Practices Badge, SBOM emit-and-attach, consumer-side `npm audit signatures` / `cosign verify` ([qf Domain 7](../research/quality-foundations.md)).
- **Repository rulesets** — protected/**immutable release tags**, signed commits, linear history, required status checks ([qf Domain 5](../research/quality-foundations.md)).

(Carried-forward and still required: the full Phase 0 gate net — `fmt`/`clippy -D warnings`/`nextest`/`cargo deny`/`cargo audit`/`tsc`/`biome`/`wasm-bindgen-test`/`publint`/`attw` — plus the Wycheproof/RFC-6962/Miri vectors activated in Phase 2.)

## Exit / freeze gate

**API FREEZE / 1.0 published.** The §11 verify pipeline exists and is byte-parity-pinned; the [§14](../research/architecture.md) surface is **clean under `cargo-semver-checks` / `cargo-public-api` / `api-extractor`**; the 1.0 trio (`thoughtmark-core 1.0.0` + `@thoughtmark/core 1.0.0` + `thoughtmark-vectors`/`@thoughtmark/vectors 1.0.0`) is published together with the toolchain pinned in `manifest.json` and verifiable provenance attached. From here on **the verbs + types + traits + the four format-identifier values are immutable except by the [§16](../research/architecture.md) rules** (additive-only `#[non_exhaustive]` variants are MINOR; a byte change is a new `canon_version` + MAJOR corpus + MINOR code, add `canon_v2` never mutate `canon_v1`; unknown version FAILS CLOSED). This is the second of the three freeze gates, following the wire-format freeze at the end of [Phase 2](phase-2-log-signing.md).

## Risks & open implementation-gaps

- **Accidental SemVer break at the freeze boundary.** Once 1.0 ships, any unintended change to the §14 surface is a covenant break. *Mitigated by* the `cargo-semver-checks` + `cargo-public-api` + `api-extractor` gates that block such a PR; additive evolution is constrained to `#[non_exhaustive]` MINOR variants with a fail-closed `_ =>` arm enforced by the §13 negative corpus.
- **Non-determinism leaking into `VerificationResult`.** A `HashMap` iteration order, an extra timestamp, or any `clock.now()` call beyond the single read at entry would break byte-parity. *Forbidden by design* ([arch §11.3](../research/architecture.md)): the result is integers / enums / constant `&'static str` / **fixed-order `Vec`s only**, no `HashMap` iteration, no timestamp other than the injected `now`; the I3 `clippy.toml` `disallowed-methods` bans `SystemTime::now`/`Instant::now`/`thread_rng`, and `verify()` takes no RNG. The `byte-parity` check fails CI if the canonical bytes diverge native vs WASM.
- **`verify()` return-convention confusion.** Returning `Err` for a tamper (instead of a value with `total = false`) would hide the honesty report. *Guarded by* the §10.3/§11.3 convention and the `verify/*` vectors that assert a tamper produces a successful run with a negative verdict and the constant `NotEstablished` block intact.
- **Reproducibility of the 1.0 artifacts.** `wasm-opt`/binaryen output is **not byte-stable across versions**, and full turnkey Rust reproducibility is still maturing upstream (`rust-lang/rust#129080`). *Mitigated by* pinning the `wasm-opt` version, building `--locked --frozen` with `SOURCE_DATE_EPOCH` + `[profile.release] trim-paths = "all"`, verifying the rebuild/`.wasm`-hash diff on **Linux CI** (where the gaps are smallest), and asserting the `manifest.json` toolchain equals CI ([qf Domain 7](../research/quality-foundations.md)).
- **Trusted-publishing setup.** OIDC trusted publishing on crates.io and npm `--provenance` must be configured before the trio can ship without long-lived tokens; the consumer-side verify step must be wired so provenance is not just produced but checked.
