# Phase 4 — Tier 2 Anchoring + Monitor

> Pin the *periodic* Merkle root to an external timebase, verify it offline, and add the split-view defenses. **Source:** [architecture.md §8](../research/architecture.md) (all subsections), [§6.6](../research/architecture.md), [§11.1–§11.3](../research/architecture.md), [§17 Phase 3](../research/architecture.md), [§18 ADR-0008/0009](../research/architecture.md), [§19](../research/architecture.md); [quality-foundations.md Domain 6](../research/quality-foundations.md) (anchoring implementation-risk flag).

## Objective

Add Tier 2 on top of the frozen 1.0 wire format and `verify()` pipeline: pin the **periodic Merkle root** (one anchor per checkpoint, *not* one per artifact, [arch §8](../research/architecture.md)) to an external timebase so that `verify()` can establish `existed_at_or_before` — an **upper** bound on existence time, read *from the proof*, never from a clock-of-now. This phase is architecture [§17 Phase 3](../research/architecture.md) ("Tier 2 anchoring"). It is a thin **async** seam entirely outside the pure core: the audited core gains only opaque types and one injected trait; all DER/CMS/OTS/Bitcoin-header parsing and all network I/O live in the `thoughtmark-anchor` family. It also lands the split-view defenses — C2SP witness cosignatures and the consistency-proof monitor ([§6.6](../research/architecture.md), [§19](../research/architecture.md)) — and wires `Policy.require_anchor` / `Policy.required_witnesses` into `verify`. Because everything is additive over a frozen API, **core stays 1.x**: each new `AnchorKind` is a MINOR release (ADR-0008, [§16](../research/architecture.md)).

## Deliverables

- **The `core::anchor` types + verifier trait are ALREADY DEFINED upstream — not redefined here** ([arch §8.2](../research/architecture.md), [§14.4](../research/architecture.md)): the opaque `AnchorReceipt { schema: "thoughtmark.anchor.receipt/v1", kind, status, root: TreeHash, checkpoint_ref: CheckpointRef, proof: Vec<u8> }`, the `#[non_exhaustive] AnchorKind { OpenTimestamps, Rfc3161, Fabric }`, `AnchorStatus { Pending { calendar_uris }, Anchored { time_lower_bound: Option<UnixMillis> } }`, `AnchorVerdict { Valid { time_lower_bound, source }, Pending, Invalid(ErrorCode) }`, `CheckpointRef { origin + tree_size + signer keyid }`, and the `AnchorVerifier` trait (`&dyn` injected into `core::verify`) were authored in [Phase 2](phase-2-log-signing.md) (the bundle's `anchors: Vec<AnchorReceipt>` field) and **frozen at the 1.0 API freeze** in [Phase 3](phase-3-verify-apifreeze.md) ([§14.4](../research/architecture.md) injection-traits-and-proof-types). Phase 4 **consumes these frozen definitions unchanged** — it does NOT re-declare them (doing so would reopen a frozen, already-compiled surface). What is new at this phase is below; the enum variants stay frozen but only now become *live* (a backend actually emits each `AnchorKind`).
  - **First concrete `AnchorKind` variants go live**: `OpenTimestamps` and `Rfc3161` get real backends + a real `AnchorVerifier` impl here (`Fabric` is deferred — see Risks). The variant *list* is frozen; landing a backend for a variant — and any later variant addition — is a MINOR (`#[non_exhaustive]`, ADR-0008, [arch §16](../research/architecture.md)).
  - I5 structural guarantee is **enforced** by the `thoughtmark-anchor` seam below: `AnchorRequest<'a> { root: &'a TreeHash, checkpoint_bytes: &'a [u8] }` ([§8.1](../research/architecture.md)) is structurally **incapable** of carrying artifact content — the only data crossing core→backend is the 32-byte root + checkpoint bytes. (`AnchorRequest` is a `thoughtmark-anchor` type, not a `core::anchor` type — it crosses the seam, never enters the audited core.)

- **`thoughtmark-anchor` seam crate** (async; never in core; ADR-0008, [arch §8.1](../research/architecture.md)/[§8.3](../research/architecture.md)/[§8.5](../research/architecture.md)):
  - The async `AnchorBackend` trait (`async-trait`): `submit` (OTS → `Pending`; RFC 3161/Fabric → `Anchored`), `upgrade` (OTS phase-2 default no-op), `refresh` (optional liveness — *not* the verifier) ([§8.1](../research/architecture.md)).
  - The **pure** `AnchorVerifier` *impl* (the concrete implementation of the trait frozen upstream in [Phase 3](phase-3-verify-apifreeze.md)) — the DER/CMS/OTS/Bitcoin-header parsers live **here**, not in the audited core — injected into `core::verify` so end-to-end verification stays offline while the audited core stays free of parsing deps ([§8.3](../research/architecture.md)). `VerifyParams { trusted_tsa_roots: &[Certificate], trusted_log_keys: &[VerifyingKey], bitcoin_headers: &dyn BlockHeaderOracle }` are all **caller-supplied** — the verifier **never fetches**.
  - `AnchorScheduler` (async, clock-injected, [§8.5](../research/architecture.md)): default `Hybrid { max_entries: 256, max_seconds: 600 }` plus an `OnDemand` flush at session end. It builds **one** signed checkpoint over the current tree and fans the **single** root to all configured backends concurrently; OTS `Pending` receipts are revisited by a background `upgrade` sweep.

- **`thoughtmark-anchor-ots`** (ADR-0009, [arch §8.4](../research/architecture.md), [Domain 6 flag](../research/quality-foundations.md)): because `rust-opentimestamps`/`opentimestamps` 0.7.x only *parses/serializes/replays* `.ots` and **cannot stamp/upgrade**, this plugin **implements the OpenTimestamps calendar HTTP protocol natively** — `submit` = `POST {calendar}/digest`; `upgrade` = `GET {calendar}/timestamp/<commitment-hex>` (200 = Bitcoin path, 404 = still pending), splicing the returned ops onto the stored timestamp. ~200 LOC over `reqwest`, no Python/Node runtime; an `ots-cli` shell-out path is kept behind a feature only as a differential-test oracle.

- **`thoughtmark-anchor-rfc3161`** ([arch §8.4](../research/architecture.md)): pure-Rust TSA client via `x509-tsp` + `tsp-asn1` (RustCrypto). Because RFC 3161 mandates a standard hash OID, when the root is SHA-256 the `messageImprint` **is the root directly**; the receipt records the imprint alg. Pin `x509-tsp` (low version) and ship real-TSA conformance vectors.

- **`thoughtmark-monitor`** ([arch §6.6](../research/architecture.md), [§19](../research/architecture.md)): the consistency-proof monitor (fetch entry bundles / checkpoints, run `verify_consistency` old→new each interval against pure core, **alarm on failure**) + a watched-participant **DID scanner** + **C2SP witness cosignatures** (a `thoughtmark-witness` reference appends its cosignature line after verifying append-only-ness). `Policy.required_witnesses` k-of-n is the **practical** split-view defense — inclusion proofs alone cannot detect a forked log.

- **Wire anchors + witnesses into `verify`** ([arch §11.1–§11.3](../research/architecture.md)): honor `Policy.require_anchor` (if true, ≥1 valid anchor required) and `Policy.required_witnesses` (k-of-n checkpoint cosignatures). `verify_anchor` (1) re-parses `checkpoint_bytes`, `verify_strict`s the signed-note signature against `trusted_log_keys`, asserts `receipt.root == checkpoint.root`; (2) dispatches per kind — **OTS:** parse `.ots`, confirm initial message == root, replay ops to a `BitcoinBlockHeaderAttestation`, look up the block via the injected `BlockHeaderOracle`, `time_lower_bound = header.timestamp` (only a remaining `PendingAttestation` ⇒ `Pending`); **RFC 3161:** verify the CMS `SignedData` chain to `trusted_tsa_roots`, confirm `TSTInfo.messageImprint` hashes to the root, read `genTime`. It **re-derives every advisory field from the opaque bytes**, never trusts a cached field, **reads time from the proof** (never a clock-of-now), and **never fetches**. The new `AnchorReceipt` check sits in the fixed order `… Consistency → AnchorReceipt (skipped if !require_anchor) → ContributionLineage` and asserts `time_lower_bound ≤ now + max_clock_skew_ms`; `Established.existed_at_or_before` is the tightest (min) upper bound over passing anchors.

## Quality gates (must be green)

These gates **activate** as the anchoring code first appears (per the activation rule), on top of the full net already standing from [Phase 0](phase-0-foundations.md):

- **Anchor receipt + real-TSA + OTS conformance vectors** ([qf Domain 6](../research/quality-foundations.md) authoritative-vectors; [Domain 8](../research/quality-foundations.md) versioned corpus): add `.ots` replay vectors and **real-TSA** RFC 3161 token vectors to `spec/vectors/` with the corpus's append-only changelog + SemVer bump; the verdict (`time_lower_bound`, `source`) is spec-anchored to the proof bytes.
- **Verify byte-parity with anchors present** (I1; [arch §13](../research/architecture.md)): the cross-language byte-identity oracle must produce a byte-identical `VerificationResult` (Rust/WASM/TS) for bundles that include `AnchorReceipt`s — the `AnchorReceipt` `CheckOutcome` and `existed_at_or_before` must match exactly.
- **`cargo-semver-checks` confirms each new `AnchorKind` is a MINOR** ([qf Domain 6](../research/quality-foundations.md) / [Domain 7](../research/quality-foundations.md)): `#[non_exhaustive]` enums keep variant additions non-breaking; core stays 1.x.
- **Fuzz target per new parser** (activation rule; ADR-0006 `fuzz/` outside the workspace): add fuzz harnesses for the **anchor proof parsers** — DER/CMS (RFC 3161 token), `.ots` op replay, and the Bitcoin block-header path — exercised against the injected `BlockHeaderOracle` so no network is touched.
- **No-fetch / offline assertion** ([arch §8.3](../research/architecture.md), [§19](../research/architecture.md)): a CI check that `verify_anchor` opens no socket and reads no clock-of-now (trust roots are caller-supplied via `VerifyParams`); the no-panic wall and `#![forbid(unsafe_code)]` continue to hold across the new parser code.

## Exit / freeze gate

`verify(bundle with anchors)` establishes **`existed_at_or_before`** — an **UPPER** bound on existence time, derived solely from the stapled proof — and reports it as a discrete `AnchorReceipt` check plus the `Established.existed_at_or_before` field, while `NotEstablished.time_upper_bound_only` keeps the honesty frame explicit. The wire-format freeze ([Phase 2](phase-2-log-signing.md)) and the API/1.0 freeze ([Phase 3](phase-3-verify-apifreeze.md)) are **not** reopened: everything here is additive. **Core stays 1.x — MINOR per new `AnchorKind`** (ADR-0008, [arch §16](../research/architecture.md)/[§17 Phase 3](../research/architecture.md)). End-to-end composition holds: `entry leaf → verify_inclusion → checkpoint root → verify_anchor → "root existed at/before T"`, with `verify_consistency` between successive anchored checkpoints proving append-only between anchors.

## Risks & open implementation-gaps

- **OTS is NOT a drop-in Rust dependency** ([qf Domain 6 flag](../research/quality-foundations.md), ADR-0009): `rust-opentimestamps` 0.7.x only parses/serializes/replays `.ots` — it does **not** stamp or upgrade. The plugin must implement the calendar HTTP protocol itself (`POST /digest`, `GET /timestamp/<hex>`, splice ops); treat anchoring as an implementation-risk gap, not a turn-key dep. Keep the `ots-cli` shell-out behind a feature as a differential oracle only (it would otherwise drag in a Python/Node runtime).
- **Fabric time is weaker than OTS/RFC 3161** ([arch §8.4](../research/architecture.md), [§19](../research/architecture.md)): the `thoughtmark-anchor-fabric` backend (chaincode keyed by root hash with an N-of-M endorsement policy — the proof's strength *is* the policy) is **deferred** here and largely lands in [Phase 7](phase-7-tier3-multiparty.md). Fabric/orderer times are orderer-asserted and **must be labelled weaker** — never presented as equivalent to OTS/RFC 3161 in any UI.
- **Inclusion proofs alone cannot detect a forked log** ([arch §19](../research/architecture.md) split-view): the only real defenses are C2SP witness cosignatures (`Policy.required_witnesses` k-of-n), the `thoughtmark-monitor` consistency checks, and the external root anchor. A verifier requiring neither witnesses nor an anchor gets a weaker guarantee — `VerificationResult.checks` must show exactly which held, and `NotEstablished` keeps the limit permanent.
- **TSA / calendar trust is caller-pinned** ([arch §8.3](../research/architecture.md), [§19](../research/architecture.md)): trust roots arrive via `VerifyParams.trusted_tsa_roots` / `trusted_log_keys` / the injected `BlockHeaderOracle`; the verifier never fetches and reads time only from the proof. Pin `x509-tsp` at a low version and lock behavior with real-TSA conformance vectors.
